Разработать продукт с названием R3 для упрощения процесса Agile.

# Детали о продукте
-  Он должен позволять дробить задачи время жизни которых (от принятия в разработку до текущего момента) больше чем заданный в настройках параметр Sprint Livecycle (2 недели по умолчанию)
- Продукт работает с уже развернутым Redmine, параметры для работы с ним так же нужно задавать в настройках
- Использовать ИИ для подсказок с разбивкой и закрытием задач. ИИ настраивается один раз на весь проект, добавить возможность использовать несколько API-ключей для распределения трат токенов (Gemini/DeepSeek/ChatGPT).
- Предлагать разбивку задачи с использованием ИИ в удобном виде (слева список текущих проблемных задач пользователя, справа кнопка для предложения вариантов разбивки (тема + описание) и под каждым вариантов кнопка создания задачи)
- Добавить вкладку Task Wizard, в которой пользователь будет разбирать свои задачи как в тиндере (закрыть/оставить текущий статус/добавить трудозатраты/перевести на другой статус) с помощью ИИ (по нажатию кнопки). ИИ должен подсказать что можно сделать с задачей (по статусу, по разбивке, по трудозатратам, по оставлению комментария)
- Оставить возможность разбивки и прочих действий без ИИ
- Интерфейс должен быть простой и современный, логотип должен быть простой
- По интерфейсу нужно оставить подсказки при наведении и прочее
- Пользователь должен иметь возможность логиниться по LDAP, но администратор все равно создается автоматически при первой развертке с кредами admin:changeme
- Настраивает работу с Redmine администратор
- Роли в системе: суперадмин, админ, продакт менеджер, пользователь
- Продакт менеджер имеет право просматривать статистику за любого пользователя и по всему беклогу проекта в целом
- Продукт должен вести историю того, как пользователи разбивают задачи
- Список задач должен быть удобным и лаконичным, можно подсвечивать критичность тянущей задачи
- Для списка нужна сортировка по дате, по протуханию, по критичности
- Так же нужно проставлять сложность задаче (будет меткой в Redmine) - s,m,l,xl,2xl и добавить кнопку для помощи в определении сложности с помощью ИИ
- Нужно считать velocity разработки и эффективность, вести учет по неделе, месяцу и дню. Субботу и воскресенье считать выходными днями

# Технические детали
- Бэкенд написан на Python FastAPI + SQLAlchemy
- База данных PostgreSQL
- Фронтенд на React с использованием какого нибудь современного UI Kit'а
- Развертка с помощью docker-compose с автоматическим созданием сертификатов для TLS (без certbot'а). Оставить возможность заходить и без TLS. На главной странице сделать кнопку скачивания корневого сертификата для последующего внесения его в доверенные центры сертификации
- Обязательные unit-тесты для всего, обязательно linter и комментарии к функциям в коде
